**Goal:** a teammate clones your repo and their AI agents pick up the project's
MCP servers and conventions with minimal setup.

The lever here is the [project source tree](/guides/projects/): a `.agentsync/`
directory committed at the repo root, with the same layout as your user
`~/.agentsync/`.

<Steps>

1. **Scaffold the project tree** at the repo root and commit it:

   ```bash
   cd myrepo
   agentsync init --scope project    # creates ./.agentsync/
   ```

2. **Set the project's agents** — an `[agents]` table (leave it empty to inherit
   each teammate's enabled agents):

   ```toml title="myrepo/.agentsync/agentsync.toml"
   [agents]
   claude   = { enabled = true }
   opencode = { enabled = true }
   ```

3. **Add the project's MCP server** as its own file, and reference team secrets
   rather than embedding them so each teammate supplies their own value locally:

   ```toml title="myrepo/.agentsync/mcp/company-api.toml"
   [server]
   type    = "stdio"
   command = "npx"
   args    = ["-y", "@company/mcp"]

   [server.env]
   COMPANY_TOKEN = "${secret:company.token}"
   ```

4. **Disable a noisy user plugin** for this repo, if needed:

   ```toml title="myrepo/.agentsync/plugins/screenshot.toml"
   [plugin]
   disabled = true
   ```

5. **Commit the whole `.agentsync/` tree** and document the one-time setup in your
   repo's README: install agentsync, set the team secret, then apply from inside
   the repo.

</Steps>

## What the teammate runs

<Steps>

1. **Install agentsync** ([install guide](/getting-started/install/)) and set the
   shared secret locally:

   ```bash
   agentsync secrets set company.token --stdin < token.txt
   ```

2. **Apply from inside the repo** — the overlay merges onto their user config:

   ```bash
   cd myrepo
   agentsync apply --dry-run
   agentsync apply
   ```

</Steps>

<Aside type="note" title="The overlay is additive">
	The project tree merges *onto* each teammate's user-level config — it doesn't
	wipe their personal servers or plugins. A project entry replaces a user entry
	with the same id/name, new entries are appended, and an empty project `[agents]`
	inherits each teammate's own enabled agents.
</Aside>
